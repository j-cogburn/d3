Universal front door. Take a natural-language request, classify intent, route to the right phase command, and run it. Replaces `/ask`.

**Usage:**
- `/d3 our signups never come back` — routed to DEFINE (a problem)
- `/d3 break OBJ-021 into solutions` — routed to SHAPE
- `/d3 run the ready work` — routed to BUILD
- `/d3 how healthy is the codebase` — routed to LEARN (git/audit)
- `/d3 where do things stand` — routed to /status + /tree

---

## Step 1 — Load context
Run `d3 index` to refresh `.d3/index.db` and surface any integrity errors. Read `.d3/vision.md`. Read `.d3/index.json` for current nodes/status.

## Step 2 — Classify intent
Map the request to one move:
- a problem / goal / "we should..." -> **/define**
- "options", "how should we", "design", an existing objective id -> **/shape**
- "build", "ship", "run ready", a ready directive id -> **/build**
- "verify", "did it work", "is it safe" -> **/prove**
- "audit", "health", "metrics", "what's true now" -> **/learn**
- "status", "where are we", "show the tree" -> **/status** then `/tree`
- "sketch/diagram X" -> **/sketch**

## Step 3 — Confirm & route
State the chosen phase in one line, then invoke that command with the user's input. If ambiguous between two, ask one short clarifying question (AskUserQuestion) before routing.
