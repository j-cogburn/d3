# Product & discovery
**Consulted at:** DEFINE · SHAPE · LEARN

- **Jobs-to-be-Done** — design for the job the user hires the product to do, not the feature requested.
- **Continuous Discovery + Opportunity Solution Tree** (Torres) — connect outcome -> opportunities -> solutions; talk to users weekly.
- **Working Backwards** (Amazon) — for big bets, write the press release + FAQ before building.
- **North Star + counter-metrics** — one metric that reflects delivered value; counter-metrics catch gaming.
- **RICE / ICE** — prioritize by reach x impact x confidence / effort.
- **Shape Up** — fixed appetite, variable scope; cut scope before extending time.

**Gate:** every objective states a measurable success metric and explicit non-goals.
