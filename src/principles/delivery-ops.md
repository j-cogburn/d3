# Delivery & ops
**Consulted at:** BUILD (--release) · PROVE · LEARN

- **DORA four-plus-one** — deployment frequency, lead time, change-failure-rate (elite 0-15%), MTTR, (+ reliability). The `/learn git` scoreboard.
- **CI/CD** — every change builds and tests headlessly.
- **Progressive delivery** — feature flags, canary 1% -> 100%; be wrong cheaply in prod.
- **SRE: SLO/SLI + error budgets** — define "good enough" numerically.
- **Observability** — logs, metrics, traces (the three pillars); OpenTelemetry.
- **Runbooks + blameless postmortems** — operations as a first-class artifact.

**Gate:** `/learn git` reports DORA; release directives use staged rollout.
